# 在 VSCode 中安装并登录 Claude Code

> 前提：已有 claude.ai 账号并订阅。

---

## 前提条件

- 已安装 VSCode
- 已有 claude.ai Pro或以上账号
- 能访问 claude.ai（需要网络代理）

---

## 第一步：安装 Claude Code 插件

1. 打开 VSCode
2. 点击左侧扩展图标（或 `Ctrl+Shift+X`）
3. 搜索框输入 **Claude Code**
4. 找到发布者为 **Anthropic** 的插件
5. 点击 **安装**

---

## 第二步：登录账号

1. 按 `Ctrl+Shift+P` 打开命令面板
2. 输入 `Claude`，选择 **Claude: Sign In**（登录）
3. 浏览器自动打开 claude.ai 授权页面
4. 点击同意授权
5. 回到 VSCode，右下角显示登录成功

---

## 第三步：开始使用

1. 左侧活动栏出现 Claude 图标，点击打开对话框
2. 打开任意代码文件，直接在对话框提问即可

---

## 常见问题

| 问题 | 原因 | 解决方法 |
|---|---|---|
| 搜索不到插件 | VSCode 未走代理 | 见下方「用代理启动 VSCode」 |
| 浏览器未自动打开 | 默认浏览器未登录 claude.ai | 手动打开浏览器，登录后重试 |
| 登录后插件无响应 | 账号未订阅 | 确认 claude.ai 订阅状态 |

---

## 附：用代理启动 VSCode（个人习惯，供参考）

VSCode 默认不读取系统代理，导致插件市场无法访问。我个人的做法是建一个 `.bat` 文件，把代理环境变量写进去，用它来启动 VSCode。

**前置：开启文件扩展名显示**

Windows 默认隐藏扩展名，改名时看不到 `.txt`，容易改错。先开启：

1. 打开任意文件资源管理器窗口
2. 点击顶部 **查看** → 勾选 **文件扩展名**

之后文件名会显示完整后缀，改名时才能正确操作。

**第一步：新建 .bat 文件**

1. 在桌面或任意位置新建一个文本文件（右键 → 新建 → 文本文档）
2. 改名为 `vscoxy.bat`（把 `.txt` 改成 `.bat`）
3. 用记事本打开，粘贴以下内容：

```bat
@echo off
set HTTP_PROXY=http://127.0.0.1:7890
set HTTPS_PROXY=http://127.0.0.1:7890
"%LOCALAPPDATA%\Programs\Microsoft VS Code\Code.exe"
```

4. 保存

> `7890` 是 Clash 默认端口，其他代理软件请替换为对应端口。  
> `%LOCALAPPDATA%` 是 Windows 自动识别的路径变量，不需要替换。

**第二步：创建桌面快捷方式**

1. 右键 `vscoxy.bat` → **发送到** → **桌面快捷方式**
2. 以后直接双击这个快捷方式启动 VSCode，代理自动生效
